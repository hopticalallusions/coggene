Methods

Literature Search

We searched for articles with abstracts or titles containing at least one genetic polymorphism terms and at least one cognitive task term. Genetic polymorphism terms consisted of any reference sequence or any gene name that occurred with at least one genetic polymorphism keyword. Task terms were collected from the authors’ knowledge, harvested from literature and screened against PubMed with the PubAtlas tool (http://www.pubatlas.org/). After combining results from a few different techniques and pruning the list based on the contents of the list of abstracts, we had a pool of 183 articles.

Functional Database Specification

We developed a strict database design to encapsulate cognitive performance scores of groups differentiated by genotype. The fundamental unit of data is a row in the Results table of the MySQL database (see Figure 1). Each row of data contains a sample size, a mean score as reported by the article and a standard deviation, either as reported by the article or converted from SEM or 95% CI. Each row links this quantitative data to pairs of gene-genotype, task-indicator, article_PMID-notes, and cognitive type with a a code number, unique to each pair. We adopted this approach (1) to attempt to standardize the stored data and (2) to allow for the addition of data in the future with minimal repetition of information.

Data Selection and Data Entry

The original pool of 183 articles was reviewed manually. We sought quantitative numerical data from healthy, human control subjects which presented data in a format compatible with our definition of a unit of data (see above). We excluded articles which provided incomplete data, solely graphical data and data from populations confounded by neuropsychiatric conditions, substance use/abuse, pharmaceutical manipulation. In a few cases, sample sizes per genotype were not given, but we were able to estimate them with the genotype frequency data, rounding the estimated values to whole numbers. Sixty-six articles met our criteria and 119 articles were excluded. Qualifying data was manually entered into a collaborative spreadsheet, and then transferred to a SQL database using custom software, providing more than 1200 units of data. To ensure reliability and double check the data entry process, two randomly selected articles were replicated independently, providing evidence that the data were entered accurately and consistently by the collaborators. Task names, indicators, gene names, genotypes and mean error types were standardized across the database by the authors. As of June 1, 2011, there were 1,711 units of data in the database.

Technical Details of the Database

The web based interface utilizes HTML5, CSS and custom JavaScript to generate dynamic plots and interactive features; it relies on the Raphaël, jQuery and FlexBox JavaScript libraries. The server side software uses Django to mediate between the web and a custom Python class which interacts with a mySQL database on a Linux based server. The front end and back end communicate with AJAX techniques, passing JSON objects from the end-user page to the server and back.

Mathematical Details of the Meta-Analysis

Source data is stored in a raw and non-normalized form. Calculation of each effect size is performed dynamically, depending on the request provided by user interactions with the web page. When the server receives a request, it retrieves all relevant ‘raw’ units of data. Each row in the raw data is compared to every other row and if the PMID (for consistent methods), task, indicator, gene, reference sequence of the two units of data are the same, and the genotypes are different, the software calculates an effect size. If those conditions are not met for a given pair of raw units of data, the data are considered non-comparable and no effect size is generated.

The ‘indiscriminate’ meta-analytic mean combines any observed effect sizes displayed on the chart, but it comes with a strong warning that the value provided is only an estimate, and should be analyzed more carefully. Users may specifically select effect sizes with the interface to obtain a better estimate, based on their judgement. The ‘stringent’ meta-analytic mean will only average effect sizes with the same task, indicator, gene, reference sequence pair and different PMIDs (in an attempt to ensure independent samples).
